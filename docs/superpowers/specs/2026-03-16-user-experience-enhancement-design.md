## 2. 功能设计

### 2.1 E. AI 功能体验增强

#### 2.1.1 家人问候

**核心概念**: AI 扮演用户的家人角色，用家乡方言风格给用户发送关怀问候。

**配置项**（用户资料页新增字段）:
| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 家人称呼 | 下拉选择 | AI 扮演的角色 | 妈妈、爸爸、奶奶、姥姥 |
| 对你的称呼 | 文本输入 | 家人怎么叫你 | 娃、闺女、儿子、小名 |
| 语气风格 | 下拉选择 | 说话风格 | 唠叨型、含蓄型、直白型、幽默型 |

**触发时机**: 用户每日签到后自动触发

**生成逻辑**:
1. 读取用户配置的家人信息
2. 读取用户家乡地点（用于方言匹配）
3. 结合当前日期、天气（可选）、时间生成问候语
4. 以家人口吻输出 20-50 字的关怀话语

**输出示例**:
> "娃啊，我是妈。看你那边今天降温了，记得加衣裳莫硬扛。吃饭莫凑合，一个人也要好好弄点吃的。钱够花不？不够给妈说。都好着呢，莫牵挂屋里。"

**数据存储**:
- 每次生成的问候记录保存到 `ai_quotes.json`
- 字段：`id`, `user_id`, `content`, `family_role`, `dialect`, `created_at`

#### 2.1.2 问候历史

**功能**: 用户可查看历次家人问候记录

**页面**: `/ai-quotes/history`
- 列表展示，按时间倒序
- 支持按家人角色筛选
- 支持收藏功能（后续迭代）

### 2.2 A. 签到流程优化

#### 2.2.1 勋章系统

**勋章类型**:

**时间里程碑系列**（基于离家天数）:
| 勋章 ID | 名称 | 条件 | 描述 |
|---------|------|------|------|
| `day_1` | 🌱 初来乍到 | 离家第 1 天 | 离开家乡的第一天 |
| `day_7` | 📅 七日维新 | 离家 7 天 | 坚持了一周 |
| `day_30` | 🌙 满月之约 | 离家 30 天 | 离家一个月了 |
| `day_100` | 🔄 百日征程 | 离家 100 天 | 百日纪念 |
| `day_180` | 💫 半年光阴 | 离家 180 天 | 半年过去了 |
| `day_365` | 🏆 一年归期 | 离家 365 天 | 离家一整年 |

**签到连续系列**（基于连续签到天数）:
| 勋章 ID | 名称 | 条件 | 描述 |
|---------|------|------|------|
| `streak_3` | 🔥 三日热 | 连续签到 3 天 | 开始上瘾了 |
| `streak_7` | ⚡ 周冠军 | 连续签到 7 天 | 坚持一周！ |
| `streak_15` | 🌟 半月谈 | 连续签到 15 天 | 半个月不间断 |
| `streak_30` | 💪 月全勤 | 连续签到 30 天 | 整月打卡 |
| `streak_100` | 👑 签到之王 | 连续签到 100 天 | 传奇！ |

**特殊成就系列**:
| 勋章 ID | 名称 | 条件 | 描述 |
|---------|------|------|------|
| `late_night` | 🌙 深夜未眠 | 凌晨 2 点后签到 | 这么晚还没睡 |
| `early_bird` | 🌅 早起鸟儿 | 早上 6 点前签到 | 起得真早 |
| `quote_master` | 📝 思乡达人 | 添加 10 条自定义话语 | 文采斐然 |

**数据存储**:
- 新增 `badges.json` 存储用户获得的勋章
- 结构：`{ user_id: [ { badge_id, earned_at } ] }`

#### 2.2.2 勋章展示

**1. 个人主页勋章墙**:
- 在仪表盘 `/dashboard` 增加"我的勋章"区域
- 网格展示已获得的勋章
- 悬停显示勋章详情（名称、获取时间、描述）
- 未获得的勋章显示为灰色锁住状态（可选）

**2. 获得时弹窗庆祝**:
- 签到后如果解锁新勋章，弹窗显示
- 动画效果：勋章放大进入 + 闪光效果
- 显示勋章名称和描述
- "确定"按钮关闭弹窗

**3. 独立勋章页面**:
- 路由：`/badges`
- 完整展示所有勋章（已获得/未获得）
- 按系列分组展示
- 显示获取进度（如"5/10"）

#### 2.2.3 签到反馈增强

**签到结果页增加内容**:
- 当前连续签到天数（醒目数字）
- 本周签到日历（7 格可视化）
- 家人问候（AI 生成）
- 新解锁勋章（如有）

### 2.3 B. 视觉和界面改进

#### 2.3.1 温馨治愈风格规范

**色彩方案**:
| 用途 | 颜色 | 色值 |
|------|------|------|
| 主色 | 暖橙色 | `#F5A623` |
| 背景 | 米白色 | `#FAF8F5` |
| 卡片背景 | 纯白 | `#FFFFFF` |
| 文字主色 | 深灰 | `#333333` |
| 文字次要 | 浅灰 | `#888888` |
| 强调色 | 淡粉色 | `#FFB6C1` |
| 成功色 | 薄荷绿 | `#98D8AA` |

**设计元素**:
- 圆角：所有卡片、按钮使用 `border-radius: 12px`
- 阴影：柔和阴影 `box-shadow: 0 2px 8px rgba(0,0,0,0.08)`
- 字体：系统字体栈，标题可用圆润字体
- 插图：手绘风格简笔画（家、月亮、邮筒、云朵）

**UI 组件规范**:
- 按钮：圆角、渐变背景、悬停轻微放大
- 卡片：白色背景、圆角、柔和阴影
- 输入框：圆角边框、焦点时柔和高亮
- 图标：线性图标、圆角端点

## 8. F. 前端 Vue 重构

### 8.1 重构策略：渐进式迁移

**原则**：
- 保持 Flask 后端模板引擎，逐步替换交互复杂的页面
- 新增功能优先用 Vue 实现
- 已有简单页面（登录/注册）暂缓迁移

**迁移顺序**：
1. 签到页面（交互最复杂）
2. 勋章墙页面（全新功能）
3. 仪表盘页面（数据展示）
4. 个人资料页（表单交互）

### 8.2 技术栈

| 技术 | 选择 | 说明 |
|------|------|------|
| 框架 | Vue 3.4+ | 使用 Composition API + `<script setup>` |
| 构建工具 | Vite 5+ | 快速 HMR，按需编译 |
| 状态管理 | Pinia | Vue 官方推荐，轻量 |
| 路由 | Vue Router 4+ | SPA 路由 |
| UI 组件 | 手写 + TailwindCSS | 灵活定制温馨治愈风格 |
| HTTP 客户端 | Axios | API 请求 |

### 8.4 API 接口规范

Flask 路由需提供 API 接口供 Vue 调用：

| 端点 | 方法 | 返回 | 说明 |
|------|------|------|------|
| `/api/checkin` | POST | `{success, checkin_date, quote, new_badges}` | 签到 |
| `/api/dashboard` | GET | `{stats, week_calendar, badges}` | 仪表盘数据 |
| `/api/badges` | GET | `[{id, name, description, earned, earned_at}]` | 勋章列表 |
| `/api/profile` | GET/PUT | 用户数据 | 个人资料 |
| `/api/ai-quotes` | GET | `[{id, content, family_role, created_at}]` | AI 问候历史 |

### 8.5 会话管理

Vue 前端通过 Cookie 继承 Flask Session：
- Flask 设置 `session` Cookie
- Vue 开发时使用 Vite 代理到 Flask，自动携带 Cookie
- 生产环境由 Flask serve 静态文件，同域部署

### 8.6 开发工作流

```bash
# 前端开发（热重载）
cd frontend
npm run dev
# Vite 代理请求到 Flask (localhost:5001)

# Flask 后端
cd ..
pixi run dev
```
